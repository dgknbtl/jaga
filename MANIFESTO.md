# 🛡️ Jaga: Vision, Mission, Manifesto

### Vision

Make context-aware security the default for HTML templating.  
Eliminate XSS at the moment code is written—not after it ships.

---

### Mission

Build a zero-dependency, ultra-lightweight engine that secures the boundary between data and the DOM—automatically, invisibly, and without slowing developers down.

---

## The Jaga Manifesto

### 1. Security is Written, Not Patched

Security shouldn’t be something you audit later.  
It should be built into the way you write code.

---

### 2. No Friction. No Excuses.

Security tools must not slow developers down.  
If it adds friction, it won’t be used—and it fails.

---

### 3. Less Code. More Protection.

Security should not cost kilobytes.  
Zero dependencies. Minimal footprint. Maximum impact.

---

### 4. Context Over Guesswork

Security is not about escaping everything.  
It’s about understanding where data goes—and acting accordingly.

---

### 5. Secure the Gaps

Frameworks protect the obvious.  
We protect what they leave behind—whether HTML, attributes, URLs, or CSS.
